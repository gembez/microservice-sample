Carsharing Mockup App

Combining Process Mining and Distributed Tracing to Improve Root Cause Analysis

The idea of the project is close the gap between the application and business layer. Therefore, a process mining technology is applied to data from a distributed tracing system that is connected to a microservice architecture. This frontend is intended for log data creation. A user (1 - 10) receives a session_ID for each session and can perform multiple activities. These activities trigger a variety of microservices in the background. When a new user is created, he or she gets a new session_ID assigned.

The activities necessary the process mining can be created by clicking on "Generate activities & reload data model". After creating the new activity log, a scheduling table is created that triggers a new data load in the process mining tool Celonis.
